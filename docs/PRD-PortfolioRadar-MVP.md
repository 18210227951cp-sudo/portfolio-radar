# 第二步：PRD（产品需求文档）— PortfolioRadar MVP

## 2.1 产品概述

- 产品名称：投组雷达（PortfolioRadar）
- 一句话描述：帮助个人投资者实时追踪持仓盈亏、可视化仓位结构的轻量分析工具

### 背景与问题
- 用户往往在多个平台开户（券商/基金平台），导致持仓与成本口径分散。
- 现有工具要么过重（社区/研究/交易闭环），要么不够可控（难自定义成本口径）。
- 用户需要一个“纯组合视角”的轻量工具：快速录入 → 自动计算 → 直观看结构与贡献度。

### 目标与边界
- 目标：让用户在 2 分钟内完成第一笔持仓录入并获得组合总览与结构图。
- 边界：MVP 不接实时行情 API，不做登录与云同步，不做复杂资产分类与税费模型。

## 2.2 核心功能列表（MVP范围）

### Must Have（必须有）

#### A. 持仓录入
- 用户可以新增一条持仓，字段包含：
  - 名称（必填）
  - 代码（可选）
  - 成本价（必填，数字）
  - 当前价（必填，数字）
  - 持仓数量（必填，数字）
- 保存后立即出现在持仓列表中并触发组合汇总与图表刷新。

#### B. 盈亏计算（自动）
- 单条持仓计算口径：
  - 当前市值 = 当前价 × 数量
  - 总成本 = 成本价 × 数量
  - 盈亏金额 = 当前市值 − 总成本
  - 盈亏比例 = 盈亏金额 / 总成本
- 组合汇总计算口径：
  - 总成本 = Σ（成本价 × 数量）
  - 总市值 = Σ（当前价 × 数量）
  - 总盈亏 = 总市值 − 总成本
  - 总收益率 = 总盈亏 / 总成本

#### C. 仓位饼图
- 展示每个持仓在组合中的资金占比：
  - 占比 = 单条市值 / 总市值
- 图例显示名称（必要时附带代码），颜色自动分配。
- 当总市值为 0 时图表显示空状态。

#### D. 组合汇总（仪表盘）
- 顶部展示 4 张卡片：
  - 总成本
  - 总市值
  - 总盈亏（红亏绿盈）
  - 总收益率（红亏绿盈）

### Should Have（应该有）

#### E. 盈亏排行榜 / 排序
- 支持按以下字段排序（至少 1 个默认排序项）：
  - 盈亏比例（降序）
  - 盈亏金额（降序）
  - 市值（降序）
- 排序影响表格展示顺序，不改变数据本身。

#### F. 本地存储（localStorage）
- 自动保存所有持仓数据到 localStorage。
- 页面刷新或重新打开时自动恢复。
- 提供“清空所有数据”入口（含二次确认）以便重置。

#### G. 删除/编辑持仓
- 删除：用户可以删除某条持仓。
- 编辑：用户可以修改名称/代码/成本价/当前价/数量，保存后实时重算与重绘。

### Could Have（可以有）

#### H. 导出 CSV
- 支持导出当前持仓列表为 CSV。
- 只导出用户录入字段与核心计算字段（可配置）。

### Won’t Have（本版不做）
- 对接真实行情 API（后续版本）
- 账户登录/云端同步（后续版本）

## 2.3 用户交互流程

1. 用户打开页面 → 看到持仓表格（默认空或示例数据）
2. 点击「添加持仓」→ 弹出输入表单
3. 填写信息后点击确认 → 持仓列表实时更新，图表重绘
4. 顶部仪表盘实时显示组合总览

### 关键交互细节（MVP）
- 输入校验：
  - 成本价/当前价/数量必须为正数（或允许 0 但有提示，取决于实现）
  - 对非法输入给出明确错误提示
- 视觉编码：
  - 盈亏为正显示绿色，为负显示红色
  - 金额格式化（千分位、小数位统一）

## 2.4 成功指标

- 用户能在 2 分钟内完成第一笔持仓录入
- 页面加载时间 < 1 秒（不含网络极差导致的 CDN 拉取异常）
- 用户在一次浏览中能回答：
  - “我整体赚/亏多少？”
  - “我的仓位主要集中在哪些标的？”
  - “哪只贡献最大/拖累最大？”

## 非功能性需求（MVP）

- 可靠性：计算口径一致、可复现；刷新不丢数据。
- 可用性：移动端与桌面端均可使用（响应式布局）。
- 可维护性：模块化函数封装，单文件内结构清晰。
